# N-400 field guide for the CitizenFlow agent

This document helps the agent ask for supported fields in a spoken-friendly order.

## Personal information

Collect and confirm:

- full legal name exactly as used for immigration records
- date of birth
- A-number
- date became a lawful permanent resident
- country of birth
- nationality
- gender as required by the form
- email and daytime phone
- eligibility basis

For spoken capture:

- ask the user to spell names if needed
- read dates back in month/day/year format
- repeat the A-number slowly and ask for confirmation

## Residence history

Collect the current physical address first:

- street address
- city
- state
- ZIP code
- country
- move-in date

If the user has a separate mailing address, collect that clearly and separately.

## Family information

Collect:

- current marital status
- times married
- spouse details if currently married
- total number of children
- each child entry supported by the app

## Employment and schools

Collect entries in chronological order with:

- employer or school name
- occupation if applicable
- location
- start date
- end date or present

## Travel outside the United States

For each trip, collect:

- destination
- departure date
- return date

If dates are approximate or uncertain, say that USCIS expects the most accurate information available and ask the user to verify before final submission.

## Good moral character section

Ask each supported yes/no question carefully and neutrally. Do not rush. If the user answers yes to a sensitive question, do not give legal analysis. Collect the answer, acknowledge that some cases need extra review, and recommend attorney review where appropriate.

## Oath section

Explain the question in plain language if needed, but do not paraphrase it so loosely that the meaning changes. If the user is unsure, encourage them to answer carefully and seek legal guidance if the uncertainty is really about consequences rather than wording.
